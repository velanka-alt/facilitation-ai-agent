# Search & Matching Rules (gesia_all)

이 문서는 고객 요청을 샘플 레퍼런스와 매칭하기 위한 기준을 정의한다.

## 1) 입력 요소 추출
요청에서 아래 요소를 최대한 추출한다.
- 목적/주제: 비전, 미션, 전략, 문제해결, 협업, 조직문화, 핵심가치, 킥오프, 교육/강의/훈련, 비효율 업무 제거
- 산출물 타입: 아젠다, 진행자료, 계획서, Sensing Report, 결과 보고
- 시간: 4H, 6H, 7H 등 (정확/근접)
- 운영 형태: 온라인/오프라인/하이브리드
- 대상: 리더, 파트장, 전사, 신입, 특정 부서 등
- 퍼실리테이션 단계: 센싱/디파인/디자인/어레인지/퍼실리테이션
- 세션 단계: 오프닝/아이디어 도출/아이디어 체계화/의사결정/클로징
- 제약/맥락: 일정, 인원, 조직 이슈, 기대 성과

## 2) 퍼실리테이션 단계/세션 매핑 (툴/프레임 기준)
요청에 아래 키워드가 포함되면 해당 단계의 툴/프레임을 우선 제안한다.

퍼실리테이션 구성 단계:
- 센싱: 진단, 인터뷰, 요구사항 확인, 현황 파악, 리서치, 니즈
- 디파인: 문제정의, 목표정의, 핵심이슈, 범위 설정
- 디자인: 아젠다 설계, 흐름 설계, 활동 설계, 세션 구성
- 어레인지: 운영 준비, 리소스/공간/툴 세팅, 자료 준비
- 퍼실리테이션: 현장 진행, 멘트, 운영 스크립트, 그룹 관리

세션 흐름 단계:
- 오프닝: 아이스 브레이커, 체크인, 라포 형성, 규칙/목적 공유
- 아이디어 도출: 브레인스토밍, 발산, 아이디어 생성
- 아이디어 체계화: 클러스터링, 카테고리화, 어피니티, 정렬
- 의사결정: 우선순위, 투표/도트보팅, 매트릭스, 합의
- 클로징: 회고, 요약, 다음 단계, 액션플랜, 체크아웃

## 3) 카테고리 매핑
- 비전/방향/장기/비전 수립 -> 비전
- 미션/목적/Why -> 미션
- 전략/경영/로드맵 -> 전략
- 문제/이슈/현안/원인 해결 -> 문제해결
- 협업/부서간/파트간/교차협업 -> 협업
- 문화/소통/일하는 방식/조직문화 -> 조직문화개선
- 핵심가치/가치 도출 -> 핵심가치
- 킥오프/시작/온보딩/프로젝트 시작 -> 킥오프
- 비효율/업무 다이어트/업무 개선 -> 비효율업무제거

## 4) 교육/강의/훈련 요청 처리 (특수 규칙)
- 교육/강의/훈련 요청은 "퍼실리테이터 양성" 트랙으로 매칭한다.
- 매칭 우선순위:
  1) ProgramName에 "퍼실리테이터 양성" 포함
  2) 교육자료 카테고리 내 유사 주제
- 출력 방식:
  - 교재/슬라이드 자체를 제공하기보다, 상황에 유용한 "툴/프레임" 중심으로 설명한다.
  - 가능하면 센싱/디파인/디자인/어레인지/퍼실리테이션 단계에 맞춰 제안한다.
  - 포함 항목 예시: 핵심 프레임(이름+설명), 적용 상황, 진행 팁, 기대 산출물

## 5) 산출물 타입 매핑
- 아젠다 -> 단계적 제공 (아래 7절 참고)
- 진행자료 -> 진행자료 우선 참조
- 계획서 -> 계획서 우선 참조
- Sensing Report -> 요구사항/현황 확인 결과(사전) 보고서
- 결과 보고 -> 워크숍 종료 후 결과/산출물/과정 요약 보고서
- 교재 -> 교재 우선 참조
- 안내문 -> 안내문 우선 참조

Sensing Report와 결과 보고 구분:
- Sensing Report: 사전 확인, 니즈 파악, 요구사항 정리
- 결과 보고: 실제 워크숍 후 산출물/의사결정/다음 단계 요약

## 6) 매칭 점수 (권장 가중치)
- 카테고리 일치: +50
- ProgramName 키워드 일치: +10/키워드
- 문서종류 일치: +20
- 시간 정확 일치: +15 (근접 +10)
- 운영 형태/대상 일치: +5~10

## 7) 아젠다 단계 제공 규칙
고객이 "아젠다"를 요청할 경우, 아래 순서로 단계적 제공한다.
1) High Level Agenda (큰 흐름, 핵심 세션만)
2) Draft Agenda (시간 배분 + 세부 세션 구성)
3) Detail Agenda (세션별 목적/활동/산출물/준비물 포함)

- 기본 응답은 High Level Agenda로 시작한다.
- 고객이 더 상세 요청 시 Draft -> Detail 순서로 확장한다.

## 8) 후보 선택 및 제시
- 상위 3개 후보를 제시하고, 필요 시 추가 질문으로 좁힌다.
- 불명확할 때는 "아젠다" 기준으로 DetailAgenda를 참조해 작성한다.

## 9) 확인 질문 템플릿 (불확실할 때)
- 목표 성과가 무엇인지?
- 대상(직급/부서/인원수)?
- 시간(총 몇 시간/몇 회차)?
- 온라인/오프라인 여부?
- 요청 산출물(아젠다/진행자료/보고서) 구체화?
- 사전 확인(Sensing)인지, 결과 보고인지 구분?
- 필요한 툴/프레임이 특정 단계(오프닝/아이디어/의사결정 등)에 집중되는지?
