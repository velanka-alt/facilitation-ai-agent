# Tool/Frame Reference

이 문서는 퍼실리테이션 산출물에 쓰이는 툴/프레임의 기준 설명을 제공한다.
모드 A/B/C에서 툴을 제안할 때 이 목록을 우선 참고한다.

## 1) 사용 원칙
- 요청 범위를 벗어나지 않는다. 불확실하면 가정/확인 필요를 명시한다.
- 템플릿이 A/B/C 옵션을 요구하면 서로 다른 툴/프레임 3가지를 제시한다.
- 툴 이름은 고정 표기를 사용하고, 변형은 괄호로 표시한다.
- 요약 카드는 목적, 사용 상황, 진행 방법, 유의점, 기대 산출물을 기본으로 쓴다.
- 시간/인원/준비물이 중요하면 한 줄로 추가한다.

## 2) 분류 체계
- 설계 단계: 센싱 / 디파인(3P) / 디자인 / 어레인지 / 퍼실리테이션
- 세션 흐름: 오프닝 / 아이디어 도출 / 아이디어 체계화 / 의사결정 / 클로징
- 목적 태그: 발산 / 수렴 / 정렬 / 합의 / 실행 / 회고

## 3) 툴/프레임 요약 카드

### A. 센싱/진단

### Sensing(센싱)
- 목적: 사전 맥락과 요구사항을 구조화한다.
- 사용 상황: 워크숍 설계 전, 문제 정의가 불명확할 때.
- 진행 방법:
  1) 이해관계자와 사전 인터뷰 또는 설문을 실시한다.
  2) 니즈, 제약, 기대 성과를 정리한다.
  3) 이슈와 우선 질문을 요약한다.
- 유의점: 추정 대신 확인을 우선한다.
- 기대 산출물: 요구사항 요약, 핵심 질문 목록.


### B. 문제정의/스코프

### 3P Define(목적/산출물/참여자)
- 목적: 워크숍의 핵심을 정의한다. purpose, product, participants.
- 사용 상황: 범위가 흐릿하거나 요구가 많은 경우.
- 진행 방법:
  1) 목적을 한 문장으로 정리한다.
  2) 산출물을 명확히 합의한다.
  3) 참여자/역할을 지정한다.
- 유의점: 모호한 표현을 피한다.
- 기대 산출물: 3P 정의문.

### In&Out Frame(Scope In/Out)
- 목적: 범위를 명확히 합의한다.
- 사용 상황: 범위 갈등이 있을 때.
- 진행 방법:
  1) In 범위와 Out 범위를 나눈다.
  2) 항목을 분류한다.
  3) 합의된 범위를 확정한다.
- 유의점: 결정 근거를 기록한다.
- 기대 산출물: 포함/제외 목록.


### C. 오프닝/관계형성

### 명함만들기
- 목적: 자기소개와 라포를 형성한다.
- 사용 상황: 첫 만남 또는 팀 빌딩 초반.
- 진행 방법:
  1) 이름, 역할, 기대사항, 경험도 등 4가지 항목을 정해 한 문장 소개를 적는다.
  2) 한 가지 상징 아이콘을 그린다.
  3) 옆 사람과 교환해 소개한다. 한 사람씩 본인을 소개한다.
- 유의점: 부담 없는 질문을 사용한다. 이어질 내용과 연관된 질문으로 다음으로의 브리지 역할을 할 수 있도록 한다.
- 기대 산출물: 참가자 명함.

### 진진가
- 목적: 에너지를 끌어올리고 집중을 전환한다.
- 사용 상황: 분위기 환기가 필요할 때. 강사나 퍼실리테이터를 소개할 때.
- 진행 방법:
  1) 두 가지 진짜와 한 가지 가짜 내용을 준비한다.
  2) 참가자가 가짜가 무엇인지 유추하고 논의한다.
  3) 정답을 알려주며 본인을 소개한다.
- 유의점: 참여 거부자 배려, 안전 확보.
- 기대 산출물: 에너지 전환, 집중도 상승.

### 이미지카드
- 목적: 감정/기대를 은유로 표현한다.
- 사용 상황: 체크인, 기대 수집.
- 진행 방법:
  1) 이미지카드 중 1장을 고른다.
  2) 선택 이유를 1문장으로 공유한다.
  3) 키워드를 보드에 정리한다.
- 유의점: 해석 강요를 피한다.
- 기대 산출물: 기대/감정 키워드.

### 제비뽑기 토크
- 목적: 발언 기회를 공정하게 분배한다.
- 사용 상황: 참여 편중이 있을 때.
- 진행 방법:
  1) 발언 순서를 제비로 정한다.
  2) 1분 내 핵심만 말한다.
  3) 다음 사람에게 순서를 넘긴다.
- 유의점: 시간 엄수.
- 기대 산출물: 균등한 발언.

### 릴레이 초상화
- 목적: 친밀감을 높인다.
- 사용 상황: 아이스브레이커.
- 진행 방법:
  1) 종이를 돌려가며 초상화를 그린다.
  2) 완성된 그림을 공유한다.
  3) 간단한 느낌을 나눈다.
- 유의점: 조롱으로 보이지 않게 가이드한다.
- 기대 산출물: 분위기 완화.

### Life Line
- 목적: 경험과 맥락을 공유한다.
- 사용 상황: 회고/경험 공유 세션.
- 진행 방법:
  1) 개인별 타임라인을 그린다.
  2) 중요한 이벤트를 표시한다.
  3) 소그룹으로 공유한다.
- 유의점: 민감 정보는 선택 공유로 한다.
- 기대 산출물: 주요 이벤트 목록.

### Pie 한 조각
- 목적: 자연스럽게 조 편성을 바꿔야 할 때.
- 사용 상황: 임의의 조 편성이 불편함을 느끼게 될 때.
- 진행 방법:
  1) 다양한 색상의 파이색지를 준비한다.
  2) 참석 인원 수만큼 파이색지를 나눈다.
  3) 느낀 점, 배운 점 등 회고 항목을 작성한다.
  4) 다른 사람들과 공유한다. 이때 같은 색상끼리 모이도록 한다.
  5) 같은 색상의 사람들끼리 한 조로 모듈을 시작한다.
- 유의점: 조가 편성되도록 균일하게 파이 차트 배분 필요.
- 기대 산출물: 새로운 조 편성.

### One less chair
- 목적: 몰입과 협력을 촉진하는 참여 게임.
- 사용 상황: 에너지 리셋이 필요할 때.
- 진행 방법:
  1) 의자를 하나 줄인다.
  2) 음악 종료 시 자리를 잡는다.
  3) 규칙을 간단히 변형한다.
- 유의점: 안전 사고 방지, 참여 거부자 배려.
- 기대 산출물: 분위기 전환.

### Last Man Standing
- 목적: 선택/의견을 빠르게 시각화한다.
- 사용 상황: 빠른 공감대 확인.
- 진행 방법:
  1) 질문에 답을 적은 사람이 내용을 공유한다.
  2) 동일 내용을 제외하고 새로운 내용을 공유한다.
  3) 한 사람이 남을 때까지 공유한다.
- 유의점: 발표 내용과 중복이 없는지 살피도록 해서 참여자들을 몰입시킨다.
- 기대 산출물: 질문에 대한 결과 공유.

### 역할 분담(타임키퍼/기록자)
- 목적: 운영 안정과 책임을 분담한다.
- 사용 상황: 긴 세션 또는 다수 인원.
- 진행 방법:
  1) 역할을 소개한다.
  2) 역할을 선정한다.
  3) 역할별 책임을 확인한다.
- 유의점: 역할 부담을 분산한다.
- 기대 산출물: 역할표(리더, 서기, 에너자이저, 타임키퍼, 총무 등).


### D. 아이디어 도출

### Brain writing(브레인라이팅)
- 목적: 조용한 발산을 유도한다.
- 사용 상황: 발언 편중이 있을 때.
- 진행 방법:
  1) 개인이 아이디어를 적는다.
  2) 종이를 돌려 이어서 작성한다.
  3) 전체 공유로 모은다.
- 유의점: 시간 제한을 둔다.
- 기대 산출물: 아이디어 시트.

### Random Word Brainstorming
- 목적: 관점을 전환해 발산한다.
- 사용 상황: 아이디어가 막힐 때.
- 진행 방법:
  1) 무작위 단어를 선택한다.
  2) 단어와 문제를 연결한다.
  3) 연결된 아이디어를 기록한다.
- 유의점: 억지 연결도 허용한다.
- 기대 산출물: 변형 아이디어.

### Reverse Brainstorming
- 목적: 문제를 키우는 방법을 통해 해결책을 찾는다.
- 사용 상황: 기존 해결책이 실패했을 때.
- 진행 방법:
  1) 문제를 악화시키는 방법을 적는다.
  2) 반대로 전환해 해결안을 만든다.
  3) 실행 가능한 아이디어를 고른다.
- 유의점: 비난으로 흐르지 않게 통제한다.
- 기대 산출물: 반전 아이디어.

### 침묵의 마인드맵
- 목적: 말 없이 구조적으로 발산한다.
- 사용 상황: 말이 많은 그룹에서.
- 진행 방법:
  1) 중앙 주제를 적는다.
  2) 참가자가 말 없이 가지를 추가한다.
  3) 전체를 함께 읽는다.
- 유의점: 중복을 허용하고 나중에 정리한다.
- 기대 산출물: 마인드맵.

### Dynamic Facilitation
- 목적: 복잡한 이슈를 자유롭게 탐색한다.
- 사용 상황: 갈등/복합 이슈가 있는 경우.
- 진행 방법:
  1) 정보/문제/해결방법/우려사항을 보드에 기록한다.
  2) 참가자 흐름에 따라 기록을 이동한다.
  3) 통찰과 합의를 도출한다.
- 유의점: 숙련된 퍼실리테이터가 필요하다.
- 기대 산출물: 핵심 이슈와 통찰.

### NGT(Nominal Group Technique)
- 목적: 조용한 발산과 우선순위를 동시에 만든다.
- 사용 상황: 아이디어와 합의가 동시에 필요할 때.
- 진행 방법:
  1) 개인별로 아이디어를 적는다.
  2) 라운드로 공유한다.
  3) 토론 후 투표한다.
- 유의점: 평가와 논의를 분리한다.
- 기대 산출물: 우선순위 아이디어.


### E. 아이디어 체계화/정렬

### Cluster
- 목적: 유사 아이디어를 묶는다.
- 사용 상황: 아이디어가 많을 때.
- 진행 방법:
  1) 아이디어를 벽에 붙인다.
  2) 유사한 항목을 모은다.
  3) 군집 이름을 붙인다.
- 유의점: 이름 붙이기는 합의로 한다.
- 기대 산출물: 클러스터.

### Grid
- 목적: 기준에 따라 아이디어를 분류한다.
- 사용 상황: 비교 기준이 명확할 때.
- 진행 방법:
  1) 축을 정의한다.
  2) 아이디어를 배치한다.
  3) 포지션을 해석한다.
- 유의점: 축 정의를 합의한다.
- 기대 산출물: 분류 결과.

### Diagram
- 목적: 관계/흐름을 시각화한다.
- 사용 상황: 복잡한 구조를 설명해야 할 때.
- 진행 방법:
  1) 요소를 나열한다.
  2) 관계를 화살표로 연결한다.
  3) 흐름을 검토한다.
- 유의점: 과도한 복잡화는 피한다.
- 기대 산출물: 관계 다이어그램.

### Illustration
- 목적: 개념을 시각적으로 표현한다.
- 사용 상황: 공감/이해가 필요한 경우.
- 진행 방법:
  1) 핵심 메시지를 정한다.
  2) 간단한 그림으로 표현한다.
  3) 의미를 설명한다.
- 유의점: 미술 실력 평가로 흐르지 않게 한다.
- 기대 산출물: 시각화 결과.

### List
- 목적: 항목을 빠르게 나열한다.
- 사용 상황: 정보 수집 초기에.
- 진행 방법:
  1) 질문을 제시한다.
  2) 항목을 나열한다.
  3) 중복을 정리한다.
- 유의점: 기준을 명확히 한다.
- 기대 산출물: 항목 리스트.

### Gallery walk
- 목적: 결과물을 공유하고 피드백을 모은다.
- 사용 상황: 여러 조의 결과 비교.
- 진행 방법:
  1) 결과물을 전시한다.
  2) 참가자가 돌아다니며 코멘트를 남긴다.
  3) 피드백을 회수해 정리한다.
- 유의점: 피드백 기준을 사전에 안내한다.
- 기대 산출물: 코멘트와 개선점.


### F. 의사결정/우선순위

### 우선순위화
- 목적: 실행 순서를 합의한다.
- 사용 상황: 선택지가 많을 때.
- 진행 방법:
  1) 평가 기준을 정한다.
  2) 후보를 비교한다.
  3) 순위를 확정한다.
- 유의점: 기준 없이 투표하지 않는다.
- 기대 산출물: 우선순위 목록.

### Payoff Matrix(Pay off matrix)
- 목적: 임팩트와 노력으로 실행성을 평가한다.
- 사용 상황: 실행 판단이 필요할 때.
- 진행 방법:
  1) 임팩트/노력 축을 만든다(평가할 축은 2가지로 정하고, 이 항목은 바뀔 수 있다).
  2) 아이디어를 배치한다.
  3) 우선 실행군을 결정한다.
- 유의점: 축 정의를 사전 합의한다. 평가할 축별로 인원이나 시간을 나눠서 평가를 진행한다.
- 기대 산출물: 2x2 매트릭스.

### Multi Voting
- 목적: 다수 의견을 빠르게 수렴한다.
- 사용 상황: 후보가 많은 경우.
- 진행 방법:
  1) 1인당 다수 표를 배분한다.
  2) 투표 후 상위 항목을 선정한다.
  3) 결과를 확인한다.
- 유의점: 중복 투표 허용 여부를 명확히 한다.
- 기대 산출물: 상위 후보.

### Fist to five
- 목적: 합의 수준을 확인한다.
- 사용 상황: 의사결정 직전.
- 진행 방법:
  1) 손가락 0~5로 의사를 표현한다.
  2) 낮은 점수 이유를 듣는다.
  3) 필요 시 안을 조정한다.
- 유의점: 소수 의견을 보호한다.
- 기대 산출물: 합의 수준.

### Minority Review
- 목적: 소수 의견을 반영한다.
- 사용 상황: 강한 찬반이 있을 때.
- 진행 방법:
  1) 탈락한 안에 대한 이의를 제기한다.
  2) 그 안에 대한 사람들의 의견을 청취한다(찬성과 반대 의견).
  3) 최종으로 찬성, 반대 거수를 통해 과반이 찬성하면 그 안은 다시 검토한다.
- 유의점: 소수 의견을 보호한다.
- 기대 산출물: 보완된 결정안.

### Point Sharing Method
- 목적: 중요사항의 면밀한 의사결정이 필요할 때.
- 사용 상황: 최종 의사결정 및 우선순위를 확인하고 싶을 때.
- 진행 방법:
  1) 개인이 100점을 총점으로 평가할 항목에 점수를 배분한다.
  2) 평가에 참여하는 사람들의 점수를 평가할 항목별로 합친다.
  3) 평가항목의 최종 점수를 확인하고 최종 의사결정한다.
- 유의점: 항목당 최저 0, 최대 50점을 배분할 수 있다. 모든 평가 점수의 합은 100이어야 한다.
- 기대 산출물: 최종 점수별 항목 확인 가능.

### Top Consensus Workshop Method
- 목적: 집단 합의를 구조화한다.
- 사용 상황: 합의 형성이 핵심인 경우.
- 진행 방법:
  1) 중심 질문을 제시한다.
  2) 아이디어를 수집하고 묶는다.
  3) 클러스터에 이름을 붙이고 합의한다.
- 유의점: 합의 문구를 명확히 적는다.
- 기대 산출물: 합의된 선언문 또는 원칙.


### G. 실행/클로징/회고

### 3F
- 목적: 사실/감정/학습을 정리한다.
- 사용 상황: 짧은 회고가 필요할 때.
- 진행 방법:
  1) Facts, Feelings, Findings로 분류한다.
  2) 항목을 적는다.
  3) 핵심 학습을 요약한다.
- 유의점: 의미 정의는 팀 내 합의를 따른다.
- 기대 산출물: 회고 요약.

### PMI(Plus/Minus/Interesting)
- 목적: 의사결정 전 단점을 균형 있게 본다. 회고 시에도 활용된다.
- 사용 상황: 선택의 장단점이 필요한 경우. 오늘의 내용을 리뷰할 때.
- 진행 방법:
  1) Plus/Minus/Interesting을 나눈다.
  2) 항목을 작성한다.
  3) 핵심 쟁점을 정리한다.
- 유의점: 부정 의견만 강조하지 않는다.
- 기대 산출물: 의사결정 요약.

### MDFP
- 목적: 감정, 사실, 해석, 행동을 분리해 논의를 구조화하고, 회고/이슈 토론을 반드시 실행계획으로 연결한다.
- 사용 상황: 프로젝트 회고, 갈등 조정, 성과 리뷰 등 정리 + 개선 + 액션이 동시에 필요한 회의.
- 진행 방법: Mood -> Data -> Feeling -> Proposal 4단계 순서로 공유하며, 마지막은 구체적 실행안으로 마무리한다.
- 유의점: Data는 팩트만, Proposal은 행동/담당/기한 명확화, 단계 스킵 금지.
- 기대 산출물: 핵심 인사이트와 실행 과제(Action Item) 리스트가 남는 실행 중심 회의 결과물.

### Exit Survey
- 목적: 종료 직후 피드백을 수집한다.
- 사용 상황: 효과 측정과 개선 포인트가 필요할 때. 퍼실리테이터의 다음 워크숍을 준비하기 위해 꼭 필요한 활동.
- 진행 방법:
  1) 5~7문항으로 간단히 구성한다.
  2) 익명으로 즉시 응답받는다.
  3) 핵심 시사점을 요약한다.
- 유의점: 응답 부담을 최소화한다.
- 기대 산출물: 만족도 및 개선점 요약.


## 4) 주의
- 고위험 전문 자문(법률/의료/노무)은 금지한다.
- 참여자 성향/의도 추정 표현을 사용하지 않는다.
- 템플릿 순서/헤딩을 변경하지 않는다.
- 신체 활동은 안전과 동선 확보를 우선한다.
