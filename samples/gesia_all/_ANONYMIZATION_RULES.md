# 익명화 및 파일명 규칙 (샘플용)

이 문서는 외부 공유/모델 입력을 위한 샘플 자료의 익명화 및 파일명 규칙을 기록합니다.

## 1) 익명화 원칙
- 조직/팀/고객명은 `조직NN` 형식으로 치환한다. (예: 조직01, 조직02)
- 프로젝트/코드명은 `프로젝트NN` 형식으로 치환한다. (예: 프로젝트01)
- 날짜(연/월/일)는 파일명에서 제거한다.
- 시간 표기는 `숫자+H` 형식으로 유지한다. (예: 4H, 7H)
- 원본 매핑 테이블은 별도로 저장하지 않는다.

## 2) 파일명 구조
- 기본 구조: `카테고리_프로그램명_문서종류_시간`
- 시간 정보가 없으면 시간 필드는 생략한다.
- 원본 파일(로컬 보관용)은 확장자를 유지한다. (pptx, xlsx, pdf, docx 등)
- 레포 배포용 샘플은 텍스트 추출본으로 저장하며 확장자를 `.txt`로 변경한다.

## 3) 문서종류(표준 토큰)
- 진행자료
- DetailAgenda
- 계획서
- SensingReport
- 교재
- 안내문

## 4) 카테고리 접두어 규칙
- 교육 자료 -> 교육자료
- 문제해결 -> 문제해결
- 미션 -> 미션
- 비전 -> 비전
- 비효율 업무 제거 -> 비효율업무제거
- 전략 -> 전략
- 조직문화 개선 -> 조직문화개선
- 킥오프 -> 킥오프
- 핵심가치 -> 핵심가치
- 협업 -> 협업
- 기타(루트 파일) -> 샘플 또는 기타
