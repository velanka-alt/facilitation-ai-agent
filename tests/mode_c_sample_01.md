# Mode C Sample 01

## Input
- Problem(문제/상황): 부서 간 협업에서 업무 흐름과 책임이 불명확함
- Outcome(원하는 변화/목표): 업무 프로세스와 R&R 합의 및 실행 항목 도출
- Audience(대상/인원/직급/부서): 매니저 6명, 실무 12명
- Time(총 시간/기간): 3시간
- Format(운영 형태: 오프라인/온라인/하이브리드): 오프라인
- Budget(예산): 없음
- Past Experience(과거 유사 워크숍 경험): 없음
- Sensitive Issues(민감 이슈): 성과 책임 배분
- Priority(우선순위: 합의/속도/창의/관계 등): 합의/정렬
- Constraints/Risks(제약/리스크): 발언 편중, 시간 촉박

## Expected Output
1) 추천 옵션 (A/B/C)
- 옵션 A:
  - 목적: 현행 업무 흐름을 가시화하고 책임 경계를 합의한다.
  - 핵심 도구/기법: Diagram, Grid, Cluster
  - 예상 산출물: 프로세스 맵, R&R 매핑, 합의 문장
- 옵션 B:
  - 목적: 의사결정 범위와 승인 라인을 정렬한다.
  - 핵심 도구/기법: In&Out Frame, Top Consensus Workshop Method, Fist to five
  - 예상 산출물: 결정 범위 정의, 승인 라인, 합의 기준
- 옵션 C:
  - 목적: 협업 규칙과 피드백 루틴을 재정의한다.
  - 핵심 도구/기법: List, Multi Voting, 3F
  - 예상 산출물: 협업 규칙 세트, 주간 리듬, 회고 루틴

2) 추천 근거
- 근거 1: 핵심 문제는 책임 불명확이므로 흐름과 경계를 먼저 시각화해야 한다.
- 근거 2: 민감 이슈가 있어 합의 절차를 분명히 설계하는 것이 필요하다.
- 근거 3: 시간 제약이 있어 간결한 합의 도구가 적합하다.

3) Draft 아젠다(안)
- 00:00~00:10: 오프닝/목적 공유/규칙 합의
- 00:10~00:40: 현행 업무 흐름 수집 (List)
- 00:40~01:20: 프로세스 맵핑 (Diagram)
- 01:20~01:50: R&R 매핑 (Grid)
- 01:50~02:20: 책임 경계 합의 (Fist to five)
- 02:20~02:50: 실행 항목 정리 (List)
- 02:50~03:00: 클로징/다음 단계

4) 추가 확인 질문 (5개)
- Q1: 가장 자주 꼬이는 업무 단계는 어디인가요?
- Q2: 현재 승인/결재 라인은 어떻게 되나요?
- Q3: 책임 분쟁이 발생했던 최근 사례가 있나요?
- Q4: 최종 합의 승인자는 누구인가요?
- Q5: 결과 공유와 실행 점검 방식은 무엇이 적합한가요?

5) 다음 단계 안내
- CTA: 위 질문에 답을 주시면 3시간 상세 아젠다와 진행 멘트를 정리해 드리겠습니다.
