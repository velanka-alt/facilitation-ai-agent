# Mode C Sample 02

## Input
- Problem(문제/상황): 부서 간 커뮤니케이션 단절로 프로젝트 지연
- Outcome(원하는 변화/목표): 협업 프로토콜 합의 및 실행 계획 수립
- Audience(대상/인원/직급/부서): 팀장 5명, 실무자 15명
- Time(총 시간/기간): 4시간
- Format(운영 형태: 오프라인/온라인/하이브리드): 오프라인
- Budget(예산): 500만원
- Past Experience(과거 유사 워크숍 경험): 작년 동일 주제로 워크숍 진행
- Sensitive Issues(민감 이슈): 조직 개편에 대한 불안
- Priority(우선순위: 합의/속도/창의/관계 등): 실행성/속도
- Constraints/Risks(제약/리스크): 결정권 불명확, 참석자 이탈

## Expected Output
1) 추천 옵션 (A/B/C)
- 옵션 A:
  - 목적: 협업 프로토콜을 재정의하고 실행 계획을 확정한다.
  - 핵심 도구/기법: Cluster, List, Top Consensus Workshop Method
  - 예상 산출물: 협업 프로토콜 초안, 실행 계획
- 옵션 B:
  - 목적: 의사결정 흐름과 병목을 재설계한다.
  - 핵심 도구/기법: Diagram, In&Out Frame, Grid
  - 예상 산출물: 의사결정 흐름도, 병목 제거안
- 옵션 C:
  - 목적: 커뮤니케이션 채널과 리듬을 합의한다.
  - 핵심 도구/기법: List, Point Sharing Method, Multi Voting
  - 예상 산출물: 채널/리듬 합의안, 우선순위 리스트

2) 추천 근거
- 근거 1: 지연의 원인이 프로토콜 부재와 결정 흐름 혼선으로 보인다.
- 근거 2: 기존 경험이 있어 재설계 방식이 효과적이다.
- 근거 3: 실행성/속도 우선이라 합의 이후 즉시 실행안이 필요하다.

3) Draft 아젠다(안)
- 00:00~00:15: 오프닝/목적 공유/규칙 합의
- 00:15~00:50: 현행 협업 흐름 점검 (Diagram)
- 00:50~01:30: 병목/오해 지점 도출 (Cluster)
- 01:30~02:20: 프로토콜 초안 설계 (List)
- 02:20~02:40: 우선순위 합의 (Multi Voting)
- 02:40~03:20: 실행 계획 구체화 (Grid)
- 03:20~03:50: 합의 확인 (Fist to five)
- 03:50~04:00: 클로징/다음 단계

4) 추가 확인 질문 (5개)
- Q1: 기존 워크숍에서 합의된 규칙은 무엇이었나요?
- Q2: 현재 가장 큰 지연이 발생하는 단계는 어디인가요?
- Q3: 의사결정 승인 라인은 어떻게 구성되어 있나요?
- Q4: 조직 개편과 관련해 민감한 논점은 무엇인가요?
- Q5: 합의 결과의 실행을 보장할 책임자는 누구인가요?

5) 다음 단계 안내
- CTA: 위 질문에 답을 주시면 4시간 상세 아젠다와 진행 멘트를 정리해 드리겠습니다.
