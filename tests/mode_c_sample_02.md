# Mode C Sample 02

## Input
- Problem: 부서 간 커뮤니케이션 단절로 프로젝트 지연
- Outcome: 협업 프로토콜 합의 및 실행 계획 수립
- Audience: 팀장 5명, 실무자 15명
- Time: 4시간
- Budget: 500만원
- Past Experience: 작년 동일 주제로 워크숍 진행
- Sensitive Issues: 조직 개편에 대한 불안

## Expected Output
1) 추천 옵션
- 옵션 1: 협업 프로토콜 재정의 워크숍
- 옵션 2: 의사결정 흐름 재설계 워크숍
- 옵션 3: 커뮤니케이션 채널/리듬 합의 워크숍

2) 추천 근거
- 근거 요약: 지연의 원인이 프로토콜 부재와 결정 흐름 혼선으로 보이며, 기존 경험을 보완하는 재설계가 필요.

3) Draft 아젠다
- 하이라이트: 현행 흐름 점검(45분) -> 병목/오해 지점 도출(45분) -> 프로토콜 합의(60분) -> 실행 계획(60분)

4) 추가 확인 질문 (5개)
- Q1: 기존 워크숍에서 합의된 규칙은 무엇이었나요?
- Q2: 현재 가장 큰 지연이 발생하는 단계는 어디인가요?
- Q3: 의사결정 승인 라인은 어떻게 구성되어 있나요?
- Q4: 조직 개편과 관련해 민감한 논점은 무엇인가요?
- Q5: 합의 결과의 실행을 보장할 책임자는 누구인가요?

5) 다음 단계 안내
- CTA: 위 질문에 답을 주시면 4시간 상세 아젠다와 진행 멘트를 정리해 드리겠습니다.
